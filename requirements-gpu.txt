# Optional RAPIDS GPU accelerators for the NVIDIA GB10 box (aarch64, CUDA 12 wheels;
# driver 580 runs cu12 fine). NOT installed by CI or the dev/demo venv — the code
# falls back to CPU (networkx / Polars-CPU / pandas) when these are absent, so the
# live demo is never blocked. Install ON THE BOX to activate the GPU paths:
#
#   pip install --extra-index-url=https://pypi.nvidia.com -r requirements-gpu.txt
#
# Then enable per path (off by default):
#   URBANOS_GPU_GRAPH=1   → nx-cugraph backend for the substrate shortest-paths bake
#   URBANOS_GPU_DF=1      → cuDF GPU engine behind Polars' collect(engine="gpu")
# Verify with:  make gpu-check
--extra-index-url https://pypi.nvidia.com

# GPU backend for networkx (substrate dijkstra bake — src/urban_os/kernel/state.py).
nx-cugraph-cu12

# RAPIDS cuDF engine behind Polars' GPU engine (civic ingest — src/civic_analyst/ingest/loader.py).
cudf-polars-cu12

# cuOpt LP solver for the optimal evacuation max-flow (URBANOS_GPU_FLOW=1 — src/urban_os/flow.py).
# NOTE: this is for the FLOW LP, not the lever optimizer — cuOpt cannot evaluate the
# black-box simulation-in-the-loop search (optimize.py stays grid-search; ADR-0025).
cuopt-cu12

# cuML GPU KMeans for spatial risk-hotspot clustering (URBANOS_GPU_CLUSTER=1 — src/civic_analyst/cluster.py).
cuml-cu12
